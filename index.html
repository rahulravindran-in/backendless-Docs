Naming Files and Directories

Backendless Functions and Directories

  • Name can only contain the following charecters: a-z or A-Z or 0-9.
  • The prescribed naming convention is Camel Casing. Example : aRandomFileName.
  • The length range of a function or it's directory is between 6 - 30 Charecters.

Backendless Logix and Directories

  • Name can only contain the following charecters: a-z or A-Z or 0-9.
  • The prescribed naming convention is Camel Casing. Example : aRandomFileName.
  • The length range of a function or it's directory is between 6 - 30 Charecters.

Action Commands

Action Commands are notation used to invoke a built-in module or feature provided for backendless logix or functions to aid in rapid development.

Asterix (*) - The Query Invoker

The asterix symbol is used to invoke a created query.

Sample Query Calls:

$variableToStoreQuery = *queryToCall( 'name' , @age );

$queryOneResult = *firstQueryToCall( 'name' , @age );
$queryTwoResult = *secondQueryToCall( @userId , @dateOfBirth );

Rules:

  1. Never nest a query call inside a query call. Support has not been given for this as it poses a readability issue, thus slowing down development speed.

    Example Wrong Implementation:

$v = *q1( *q2( ) );

    Example Correct Implementation:

$a = *q1( @name ); 
$v = *q2( $a );

  2. Leave space between bracket and variable names inside the invoking query. 

    Example Bad usage:

$v = *q1(@a,@b);

    Example Good usage:

$v = *q1( @a , @b );
